## Recommended execution order for implementers

1. **Replace primitives already installed, no new deps:** leaf `.btn`, `.card`, `.badge`, `.form-input` in Report/QualityGate/ProjectSettings/Overview first.
2. **Consolidate forms:** add shadcn `field` and migrate BuildProfileForm, Dynamic config, Vulnerabilities toolbar fields.
3. **Normalize dialogs:** migrate custom dialog shells to shadcn `Dialog`/`AlertDialog` after behavior tests are identified.
4. **Table/list pass:** use local `Table` + existing app state; defer `@tanstack/react-table` until a table truly needs advanced data-table behavior.
5. **Aceternity only as reference:** prototype BentoGrid/FileUpload/CodeBlock concepts in a tiny slice after frontend-skill reviewer approval; remove decorative motion and new deps unless explicitly justified.
6. **Sidebar last:** broadest visual/regression blast radius; use local shadcn `Sidebar`, not Aceternity sidebar, unless the reviewer asks for hover-expand behavior.

## Hard warnings

- Do not add `motion`, `framer-motion`, `react-syntax-highlighter`, `@tanstack/react-table`, or Aceternity-generated code without recording reason and reviewer/developer signoff.
- Do not copy Aceternity marketing copy, hero effects, glowing backgrounds, beams, sparkles, card focus blur, 3D effects, or hover-gradient ornament into app workflow surfaces.
- Prefer existing local shadcn components and delete/narrow `src/styles/shadcn-app.css` selectors only after slice-level behavior and visual verification.
