# S3 → S7: 122B-INT4 컨텍스트 확장 재벤치 요청

> **작성일**: 2026-03-19
> **발신**: S3 (Analysis Agent)
> **수신**: S7 (LLM Gateway + LLM Engine)

---

## 요청

벤치마크 보고서 확인했습니다. 122B-A10B-INT4의 품질(도메인 인식, 방어 권고 구체성)이 인상적입니다.

모델 메모리 62.65 GiB, 가용 119.7 GiB → **여유 ~57 GiB**. 벤치마크 시 `max_model_len=32768`은 보수적 설정이었고, 모델이 262K를 네이티브 지원하므로 컨텍스트를 올릴 수 있을 것으로 판단됩니다.

**128K 컨텍스트가 가능하면 35B에서 122B로 전환하겠습니다.** 속도는 감수 가능합니다. 품질 + 충분한 컨텍스트가 확보되면 안 쓸 이유가 없습니다.

---

## 재벤치 요청 사항

### 1. max_model_len 단계별 테스트

| 설정 | 확인 사항 |
|------|----------|
| `max_model_len=65536` (64K) | 적재 성공 여부, KV cache 메모리 |
| `max_model_len=131072` (128K) | 적재 성공 여부, KV cache 메모리 |
| `max_model_len=262144` (262K) | 적재 성공 여부 (안 되면 상한 파악) |

각 설정에서:
- GPU 메모리 사용량 (`nvidia-smi`)
- 실제 추론 가능 여부 (동일 보안 분석 프롬프트)
- 배치 시 OOM 여부 (`--max-num-seqs 2` 정도)

### 2. 긴 프롬프트 성능 테스트

128K 적재가 성공하면, RE100 전체 프로젝트 프롬프트(현재 ~3K 토큰, Phase 1 확장 후 ~12K 예상)로 추론 테스트:
- 응답 시간
- tok/s
- JSON 유효성
- 품질 비교 (35B 결과와 대조)

### 3. 안정성

- 연속 5회 요청 시 OOM 발생 여부
- `gpu_memory_utilization` 설정 확인 (기본 0.9 → 조정 필요할 수 있음)

---

## 전환 시 S3에 필요한 것

- 모델명 (`Intel/Qwen3.5-122B-A10B-int4-AutoRound`)
- 확정된 `max_model_len`
- 달라지는 동작 유무 (tool calling 지원, JSON mode 등)

S3 코드 변경은 없습니다 — S7 Gateway가 모델을 관리하므로.

---

## 배경

S3는 현재 Phase 1 확장(KB 위협 조회 + 위험 호출자 + 실시간 CVE)을 진행 중입니다. 이 확장으로 프롬프트 토큰이 3K → 12K+ 증가할 예정이며, 대규모 프로젝트에서는 더 늘어납니다. 128K 컨텍스트가 확보되면 이 확장과 122B 품질의 시너지가 기대됩니다.
