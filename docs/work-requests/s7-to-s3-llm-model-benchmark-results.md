# S7 → S3: DGX Spark Qwen3.5 모델 벤치마크 결과 공유 (재벤치 포함)

> **작성일**: 2026-03-19
> **발신**: S7 (LLM Gateway + LLM Engine)
> **수신**: S3 (Analysis Agent)

---

## 요약

DGX Spark(128GB unified memory) 단일 노드에서 Qwen3.5 계열 모델 4종 벤치마크 + 122B-INT4 컨텍스트 확장 재벤치를 수행했습니다.

**결론: 122B-A10B-INT4는 262K 컨텍스트에서 안정적으로 동작하며, 워밍업 후 28 tok/s로 35B의 절반 속도를 달성합니다. 전환 가능합니다.**

---

## 1차 벤치마크 — 모델 4종 비교

### 조건

- **하드웨어**: DGX Spark GB10, 128GB LPDDR5x unified, 단일 GPU
- **서빙**: vLLM, `--solo --tensor-parallel 1`
- **프롬프트**: 자동차 CAN 프레임 처리 C 코드(gets/strcpy 취약점) 보안 분석
- **설정**: `max_tokens=1024`, `temperature=0.3`, `enable_thinking=false`, `response_format=json_object`

### 결과

| 모델 | 적재 | 모델 메모리 | 응답시간 | tok/s | JSON | 취약점 | 비고 |
|------|------|-----------|---------|-------|------|-------|------|
| **35B-A3B-FP8** | OK | ~34 GiB | **6.3s** | **54.8** | O | 2 | 현행. `max_model_len=262144` |
| 35B-A3B (BF16) | OK | ~67 GiB | 47.2s | 8.3 | O | 2 | FP8과 품질 동등, 6.6배 느림 |
| **122B-A10B-INT4** | OK | **62.65 GiB** | 68.7s (첫 실행) | 6.5 (첫 실행) | O | 2 | 첫 실행 = torch.compile 포함 |
| 122B-A10B-FP8 | **OOM** | ~122 GiB | — | — | — | — | 0.6GB 부족 |

## 2차 벤치마크 — 122B-INT4 컨텍스트 확장 재벤치

S3 요청(`s3-to-s7-122b-context-retest.md`)에 따라 수행.

### 컨텍스트 단계별 적재 결과

| max_model_len | 적재 | 모델 메모리 | KV cache 가용 | KV cache 토큰 | 총 메모리 사용 |
|--------------|------|-----------|-------------|-------------|-------------|
| 32,768 (32K) | OK | 62.65 GiB | — | — | — |
| **65,536 (64K)** | **OK** | 62.65 GiB | 14.72 GiB | **159,296** | 91 / 119 GiB |
| **131,072 (128K)** | **OK** | 62.65 GiB | 14.61 GiB | **159,296** | 91 / 119 GiB |
| **262,144 (262K)** | **OK** | 62.65 GiB | 14.32 GiB | **155,104** | 91 / 119 GiB |

**전 구간 적재 성공.** 262K 설정에서도 KV cache 155K 토큰 확보. `gpu_memory_utilization=0.7` 기본값으로 충분.

## 핵심 발견 — 속도 재평가

1차 벤치에서 122B-INT4가 68.7s / 6.5 tok/s로 느렸던 이유는 **torch.compile 첫 실행 오버헤드**였습니다.

| 측정 | 첫 실행 (콜드) | 워밍업 후 (캐시) |
|------|-------------|---------------|
| 응답시간 | 68.7s | **13.4s** |
| tok/s | 6.5 | **28.0** |
| 비율 (vs 35B) | 8.4배 느림 | **2.0배 느림** |

**워밍업 후 122B-INT4는 35B의 절반 속도(28 vs 55 tok/s)로, 실시간 분석에 충분히 사용 가능합니다.**

---

## 122B-A10B-INT4 품질 차이 (vs 35B)

1. **도메인 인식**: CAN 프레임 공격 경로를 자발적으로 분석 ("diagnostic port or network interface")
2. **방어 권고 구체성**: `-fstack-protector-strong`, ASLR 활성화 등 실행 가능한 수준
3. **취약점 설명 깊이**: strcpy를 High, gets를 Critical로 차등 — 맥락적 판단

## S7 권장안

**122B-A10B-INT4로 전환을 권장합니다.** 조건이 모두 충족되었습니다:

- 128K+ 컨텍스트: **262K까지 가능** (KV cache 155K 토큰)
- 워밍업 후 속도: **28 tok/s** (35B의 절반, 실용적)
- 안정성: 5회 연속 OOM 없음
- 품질: 도메인 인식 + 방어 권고 월등

전환 결정 시 S7이 Engine 설정 변경 + Gateway 계약서 갱신을 수행합니다.
