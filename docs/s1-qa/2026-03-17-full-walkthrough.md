# QA 피드백 — 전체 페이지 훑어보기 (2026.03.17)

## 환경
- URL: http://localhost:5173
- 브라우저: Chrome
- 프로젝트: 정적 분석 테스트

## 전체 소감

전반적으로 사이드바 네비게이션, 페이지 라우팅, 데이터 렌더링 모두 안정적이다. 각 페이지 간 이동 시 크래시나 빈 화면 없이 정상 전환되며, 빈 상태(동적 분석, 동적 테스트, 보고서)에서도 적절한 안내 문구와 CTA 버튼이 표시된다.

정적 분석 페이지의 대시보드는 2탭 구조(최신 분석 / 전체 현황)가 잘 동작하며, Quality Gate 배너, StatCard, 도넛 차트, 취약 파일 Top 5 등이 기대대로 렌더링된다.

다만 몇 가지 눈에 띄는 점이 있어 아래 정리한다.

## 이슈 목록

### [Minor] 브레드크럼 한/영 불일치
- **위치**: 분석 이력 페이지, 보고서 페이지 — 상단 브레드크럼
- **재현**: 사이드바에서 "분석 이력" 클릭, 또는 "보고서" 클릭
- **기대 동작**: 브레드크럼에 한글로 "분석 이력", "보고서" 표시
- **실제 동작**: "analysis-history", "report"로 영문 slug가 그대로 노출됨. 다른 페이지(정적 분석 → "정적 분석", 프로젝트 설정 → "프로젝트 설정" 등)는 한글로 정상 표시되어 불일치가 있다.

### [Major] 보고서 페이지 — 분석 결과 있는데 "분석을 먼저 실행해주세요" 표시
- **위치**: 보고서 페이지
- **재현**: 사이드바에서 "보고서" 클릭
- **기대 동작**: 정적 분석 Run이 1건 완료된 상태이므로 보고서 생성이 가능하거나, 최소한 "정적 분석 결과 기반 보고서 생성" 등의 안내가 표시되어야 한다.
- **실제 동작**: "보고서를 생성할 수 없습니다. 분석을 먼저 실행해주세요"로 표시됨. 실제로 정적 분석 Run이 존재하므로 조건 판단이 올바르지 않은 것으로 보인다.
- **콘솔 에러**: 이 페이지 접근 시 API 500 INTERNAL_ERROR 2건 발생 (`client.ts:95:12`)

### [Minor] 취약점 목록 — Finding 수 불일치 (52건 vs 65건)
- **위치**: 취약점 목록 페이지 헤더 vs 정적 분석 최신 분석 탭
- **재현**: 취약점 목록 페이지 접근
- **기대 동작**: 취약점 총 건수가 일관되게 표시
- **실제 동작**: 취약점 목록 페이지 상단에 "총 52건"으로 표시되지만, 바로 아래 정적 분석 Run 카드에는 "65건"으로 표시됨. 정적 분석 최신 분석 탭의 Finding StatCard도 65. 도넛 차트는 52로 표시. 52는 활성(active) Finding 수이고 65는 Info 포함 전체인 것으로 추정되나, 사용자 입장에서 혼동될 수 있다. 표기 기준이 명확하지 않다.

### [Suggestion] Overview 페이지 — StatCard의 심각도별 수치 합계
- **위치**: Overview 페이지 > StatCard 영역
- **재현**: Overview 페이지 접근
- **실제 동작**: Critical 6 + High 10 + Medium 30 + Low 6 = 52인데, 도넛 차트도 52 취약점으로 표시. 정적 분석 페이지에서는 Finding 65건, 도넛 차트 52건으로 나뉘어 있다. Info(13건)가 포함/미포함 기준이 페이지마다 다르게 적용되는 것으로 보인다. 어디에 Info가 포함되고 어디에 제외되는지 일관된 안내 또는 라벨이 있으면 좋겠다.

### [Suggestion] 설정 페이지 — 사이드바에 "설정" 메뉴 없음
- **위치**: 프로젝트 내 사이드바
- **재현**: 프로젝트 진입 후 사이드바 확인
- **기대 동작**: QA 가이드에 "설정" 사이드바 메뉴가 있다고 명시됨
- **실제 동작**: 사이드바 메뉴 목록에 "설정"이 별도로 보이지 않고, 사이드바 최하단의 톱니바퀴 아이콘(설정)을 클릭해야 접근됨. 기능적으로는 문제없으나, 가이드 문서와 실제 UI가 약간 다르다.
