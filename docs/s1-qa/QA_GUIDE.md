# QA 가이드 — Smartcar Security Framework (Frontend)

> 이 문서는 QA 담당자가 테스트 전에 반드시 읽어야 할 가이드입니다.

---

## 1. 프로젝트 개요

차량 보안 분석 프레임워크의 **데스크탑 운영 콘솔** (Electron + React).
소스코드의 정적 분석, CAN 통신 동적 분석, 퍼징 테스트 결과를 보여주고 triage하는 도구.

- **접속 URL**: `http://localhost:5173`
- **프로젝트 진입**: 좌측 사이드바에서 프로젝트 선택 → 각 페이지로 이동

---

## 3. 정적 분석 페이지 상세 구조

### 3.1 대시보드 (기본 화면)

2개 탭으로 구성:

**"최신 분석" 탭 (기본 선택됨)**
- Quality Gate 배너 (통과/실패/경고)
- StatCard 3개: Finding 수, Critical+High 건수, 소요 시간
- 심각도 도넛 차트 + 취약 파일 Top 5
- Finding 목록 (파일별 그룹)

**"전체 현황" 탭**
- 기간 선택기 (7일/30일/90일/전체)
- StatCard 4개: 총 Finding, 미해결, Gate 통과율, Run 수
- 심각도/출처 분포 차트, 트렌드, 상태 분포
- 파일/룰 랭킹, 최근 Run 목록

### 3.2 주요 플로우

```
대시보드 → [새 분석] → 모드 선택 → 분석 진행 → 완료 → 대시보드 자동 갱신
대시보드 → Finding 클릭 → Finding 상세 → [뒤로] → 대시보드
대시보드 → 전체 현황 탭 → 최근 Run 클릭 → Run 상세 → Finding 클릭 → Finding 상세
```

## 4. 테스트 시 주의사항

### 반드시 확인

- **탭 전환**: 최신 분석 ↔ 전체 현황 전환 시 데이터 유지되는지
- **네비게이션 복귀**: Finding 상세에서 뒤로가기 → 대시보드로 정상 복귀하는지
- **빈 상태**: 완료된 Run이 없을 때 "최신 분석" 탭에 적절한 안내 표시되는지
- **활성 분석 배너**: 분석 진행 중일 때 배너가 탭과 무관하게 항상 보이는지
- **토스트 메시지**: 경고/에러 시 우측 하단에 토스트 표시되는지 (3초 후 자동 닫힘)
- **취약 파일 클릭**: Top 5 파일 클릭 시 파일 상세로 이동하는지, "기타" 클릭 시 토스트 경고 나오는지

### 알려진 이슈 (S2 처리 대기)

아래는 백엔드 버그로, 프론트 테스트 시 참고만 하고 이슈로 올리지 않아도 됨:

- **증적 175개 표시**: Finding 상세에서 증적(Evidence)이 비정상적으로 많음 — S2에 수정 요청 완료
- **소요 시간 0초**: Run의 startedAt/endedAt 타임스탬프 버그 — S2에 수정 요청 완료
- **Finding 위치 "기타"**: location이 특정되지 않아 모든 Finding이 "기타"로 묶임 — S2 확인 중

### 테스트 범위 밖

- 동적 분석/동적 테스트: 어댑터(하드웨어) 연결이 필요하여 현재 테스트 불가
- 백엔드 API 응답 내용의 정확성: S2 담당

## 5. 피드백 문서 작성 규칙

### 파일명

```
{날짜}-{테스트대상}.md
예: 2026-03-17-static-dashboard.md
```

### 심각도 분류

| 등급 | 기준 |
|------|------|
| **Critical** | 페이지 크래시, 기능 완전 불능, 데이터 유실 |
| **Major** | 기능 동작하지만 결과가 틀림, 네비게이션 깨짐 |
| **Minor** | UI 깨짐, 텍스트 잘림, 정렬 어긋남 |
| **Suggestion** | 개선 제안, UX 불편 사항 |

### 이슈 작성 형식

```markdown
### [{심각도}] 이슈 제목

- **위치**: 페이지명 > 탭명 > 컴포넌트명
- **재현**: 1. xxx 2. xxx 3. xxx
- **기대 동작**: xxx
- **실제 동작**: xxx
- **콘솔 에러**: (있으면 복사)
```

### 주의

- **스크린샷 ID나 상태 묘사**를 최대한 상세히 적어주세요 — 개발자는 브라우저를 열 수 없습니다
- 같은 원인의 이슈는 묶어서 하나로 작성
- "알려진 이슈"에 해당하는 건 제외
