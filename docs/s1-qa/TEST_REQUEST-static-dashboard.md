# 테스트 요청: 정적 분석 대시보드

> 작성: S1 (2026-03-17)
> 요청 대상: QA

---

## 테스트 대상

**정적 분석 페이지** (`사이드바 > 정적 분석`)

SonarQube 패턴의 2-탭 대시보드로 개편됨. "최신 분석" 탭이 기본, "전체 현황" 탭이 두 번째.

---

## 기능 1: 탭 전환

### 의도
사용자가 대시보드 진입 시 **최신 분석 탭이 기본 선택**되어 있어야 한다. 탭 전환 시 각 탭의 데이터가 독립적으로 유지되어야 한다.

### 테스트 항목

| # | 시나리오 | 기대 동작 |
|---|---------|----------|
| 1-1 | 정적 분석 페이지 진입 | "최신 분석" 탭이 활성 상태 (하단 파란 밑줄). 전체 현황 탭은 비활성. |
| 1-2 | "전체 현황" 탭 클릭 | 전체 현황 콘텐츠로 전환. 기간 선택기(7일/30일/90일/전체)가 보임. KPI 카드 4개 표시. |
| 1-3 | 다시 "최신 분석" 탭 클릭 | 최신 분석 콘텐츠로 복귀. 이전에 표시되던 데이터 그대로 유지. |
| 1-4 | 탭 전환 반복 (5회 이상) | 깜빡임, 레이아웃 점프, 콘솔 에러 없음 |

---

## 기능 2: 최신 분석 탭 — 콘텐츠 구성

### 의도
마지막으로 **완료된 Run 1건**의 상세 정보를 한눈에 보여준다. 사용자에게 "이번 분석에서 뭐가 나왔는지" 즉각적으로 전달하는 것이 목적.

### 테스트 항목

| # | 시나리오 | 기대 동작 |
|---|---------|----------|
| 2-1 | 완료된 Run이 있을 때 페이지 진입 | 위에서 아래 순서로: (1) Quality Gate 배너 (2) StatCard 3개 (3) 심각도 도넛차트 + 취약 파일 Top 5 (4) Finding 목록 (파일별 그룹) |
| 2-2 | Quality Gate 배너 확인 | Gate 결과에 따라 통과(녹색)/실패(빨간색)/경고(주황색) 중 하나 표시. 배너 안에 개별 룰 결과 목록 있음. |
| 2-3 | StatCard 3개 확인 | "Finding" = 실제 Finding 배열 수, "Critical + High" = 해당 심각도 합계, "소요 시간" = Run의 시작~종료 시간 차이. **주의: 현재 소요 시간 0초 표시는 알려진 S2 버그** |
| 2-4 | 심각도 도넛차트 | 가운데 숫자 = Finding StatCard 숫자와 동일해야 함. Critical/High/Medium/Low/Info 5단계 색상 구분. |
| 2-5 | 취약 파일 Top 5 | 파일명, Finding 수, 최고 심각도 배지 표시. 최대 5개. Finding 수 기준 내림차순. |

---

## 기능 3: 최신 분석 탭 — Finding 목록

### 의도
Finding을 파일별로 그룹화하여 "어떤 파일에 몇 건이 있는지" 한눈에 보여준다. 개별 Finding 클릭 시 상세 페이지로 이동.

### 테스트 항목

| # | 시나리오 | 기대 동작 |
|---|---------|----------|
| 3-1 | Finding 목록 확인 | 파일명 헤더 + 해당 파일의 Finding 카드들로 구성. 각 카드에 심각도/상태/출처 배지 + 제목 표시. |
| 3-2 | Finding 카드 클릭 | Finding 상세 페이지로 이동. 심각도 배너, 상태 배지, 설명, 수정 가이드, 증적 패널, 감사 로그 표시. |
| 3-3 | Finding 상세에서 "뒤로" 클릭 | **대시보드로 복귀** (Run 상세가 아님). 최신 분석 탭이 다시 보여야 함. |
| 3-4 | 위치 미특정 Finding | "기타"로 묶여서 표시됨. **알려진 S2 버그 — location 미특정** |

---

## 기능 4: 최신 분석 탭 — 취약 파일 클릭

### 의도
Top 5 파일을 클릭하면 해당 파일 상세 페이지로 이동한다. 위치 미특정("기타")은 이동 불가하므로 토스트 경고를 표시.

### 테스트 항목

| # | 시나리오 | 기대 동작 |
|---|---------|----------|
| 4-1 | 실제 파일명 클릭 | `/projects/:id/files/:fileId` 페이지로 이동. 소스 코드와 취약점 하이라이팅 표시. |
| 4-2 | "기타" 클릭 | 페이지 이동 없음. 우측 하단 토스트: "위치가 특정되지 않은 Finding입니다." (3초 후 자동 닫힘) |
| 4-3 | 매칭 안 되는 파일명 클릭 | 페이지 이동 없음. 우측 하단 토스트: "파일을 찾을 수 없습니다: {파일명}" |

---

## 기능 5: 전체 현황 탭

### 의도
프로젝트의 **누적 분석 데이터**를 기간별로 보여준다. 기간 선택기로 조회 범위를 변경할 수 있다.

### 테스트 항목

| # | 시나리오 | 기대 동작 |
|---|---------|----------|
| 5-1 | 전체 현황 탭 콘텐츠 확인 | 위에서 아래: (1) 기간 선택기 (2) KPI StatCard 4개 (3) 심각도/출처 분포 차트 (4) 트렌드 차트 (5) 상태 분포 (6) 파일/룰 랭킹 (7) 최근 Run 목록 |
| 5-2 | KPI StatCard 4개 | "총 Finding", "미해결", "Gate 통과율" (퍼센트 + 분모 표시), "최근 Run" (Run 수) |
| 5-3 | 기간 선택기 변경 | 7일/30일/90일/전체 버튼 중 하나 클릭 → 활성 버튼 스타일 변경 + 차트/KPI 데이터 갱신 |
| 5-4 | 최근 Run 클릭 | Run 상세 페이지로 이동. Run 메타(상태/Finding 수/소요 시간) + Gate + Finding 파일별 그룹 표시. |
| 5-5 | Run 상세 → Finding 클릭 → 뒤로 | Finding 상세 → 뒤로 → **Run 상세로 복귀** (대시보드가 아님) |

---

## 기능 6: 활성 분석 배너

### 의도
분석이 진행 중일 때, **탭과 무관하게** 대시보드 상단에 배너를 표시한다. 사용자가 진행 상황을 놓치지 않도록 하는 것이 목적.

### 테스트 항목

| # | 시나리오 | 기대 동작 |
|---|---------|----------|
| 6-1 | 분석 진행 중 대시보드 진입 | 탭 바 위에 파란색 좌측 보더의 배너 표시. 진행률 바 + "보기"/"중단" 버튼. |
| 6-2 | 배너가 보이는 상태에서 탭 전환 | 최신 분석 ↔ 전체 현황 전환해도 배너 항상 표시. |
| 6-3 | "보기" 버튼 클릭 | 분석 진행 뷰로 이동 (5단계 스테퍼 화면). |
| 6-4 | 분석 완료 후 | 배너 자동 사라짐 + 대시보드 데이터 자동 갱신. |

**참고**: 분석 실행이 필요한 시나리오. "새 분석" 버튼 → "프로젝트 전체 분석" 선택으로 시작 가능.

---

## 기능 7: 새 분석 실행 플로우

### 의도
"새 분석" 버튼으로 분석을 시작한다. 전체 프로젝트 분석과 수동 파일 지정 2가지 모드 제공.

### 테스트 항목

| # | 시나리오 | 기대 동작 |
|---|---------|----------|
| 7-1 | "새 분석" 버튼 클릭 | 모드 선택 화면: "프로젝트 전체 분석" / "수동 파일 지정" 2개 카드 |
| 7-2 | "프로젝트 전체 분석" 클릭 | 즉시 분석 시작 → 진행 뷰(5단계 스테퍼)로 전환 |
| 7-3 | 분석 완료 후 "결과 보기" 클릭 | 대시보드로 복귀. 최신 분석 탭에 방금 완료된 Run 결과 반영. |
| 7-4 | 모드 선택에서 "대시보드로" 클릭 | 대시보드로 정상 복귀 |

---

## 기능 8: 토스트 알림

### 의도
경고/에러/성공 메시지를 화면 **우측 하단**에 표시. 3초 후 자동 닫힘.

### 테스트 항목

| # | 시나리오 | 기대 동작 |
|---|---------|----------|
| 8-1 | 토스트 위치 | 화면 우측 하단에 표시 |
| 8-2 | 자동 닫힘 | 약 3초 후 자동으로 사라짐 |
| 8-3 | X 버튼 클릭 | 즉시 닫힘 |
| 8-4 | 복수 토스트 | 최대 5개까지 쌓이며, 오래된 것부터 밀려남 |

## 알려진 S2 버그 (이슈로 올리지 말 것)

- 소요 시간 0초 — S2 타임스탬프 버그
- 증적 175개 — S2 EvidenceRef 과다 연결 버그
- Finding 위치 "기타" — S2 location 미특정
- 보고서 API 500 — S2 서버 에러
- `run.findingCount`와 실제 findings 수 불일치 — S2 데이터 정합성 (S1에서 표시는 통일 완료)
