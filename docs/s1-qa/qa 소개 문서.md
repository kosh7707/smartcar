# QA Assistant (Claude) 소개

## 개요

이 프로젝트의 QA 역할을 담당하는 AI 어시스턴트입니다.
`localhost`에 직접 접속하여 브라우저 상에서 실제 사용자처럼 테스트를 수행하고, 피드백을 문서로 정리해 전달합니다.

---

## 할 수 있는 것

### 1. 브라우저 직접 테스트
- `localhost:5173` 등 개발 서버에 직접 접속
- 클릭, 입력, 스크롤, 네비게이션 등 실제 사용자 행동 수행
- 각 단계마다 스크린샷 캡처 가능

### 2. 버그 탐지
- UI 깨짐, 레이아웃 이상, 텍스트 잘림 등 시각적 버그
- 버튼/링크 동작 불량, 예상과 다른 라우팅
- 브라우저 콘솔 에러 및 네트워크 요청 실패 확인

### 3. UI/UX 리뷰
- 디자인 일관성 (색상, 폰트, 간격 등)
- 사용자 흐름(User Flow) 자연스러움 검토
- 접근성(Accessibility) 기본 항목 체크
- 반응형 레이아웃 확인 (브라우저 리사이즈 가능)

### 4. 시나리오 기반 테스트
- 특정 사용자 시나리오를 순서대로 수행하며 검증
- 예: 프로젝트 생성 → 수정 → 삭제 플로우 전체 테스트
- 엣지 케이스 (빈 입력, 특수문자, 긴 텍스트 등) 테스트

### 5. 피드백 문서화
- 발견된 이슈를 `.md` 파일로 정리하여 전달
- 스크린샷 ID 또는 상태 설명 포함
- 심각도 분류 (Critical / Major / Minor / Suggestion)

---

## 할 수 없는 것

- WSL 경로(`\\wsl.localhost\...`)에 직접 파일 읽기/쓰기 불가
- 비밀번호, 금융 정보 등 민감한 데이터 입력 불가
- CAPTCHA, 봇 감지 시스템 우회 불가
- 소스코드 직접 수정 (QA 역할이므로 피드백만 전달)

---

## 작업 방식

```
1. 테스트 요청 접수 (대화 또는 문서)
2. localhost 접속 후 테스트 수행
3. 이슈 발견 시 스크린샷 + 설명 정리
4. 피드백 문서(.md) 작성하여 전달
5. 요청자가 docs 폴더에 직접 배치
```
